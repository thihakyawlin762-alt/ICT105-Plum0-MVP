# Team Working Agreement

1. **Meeting Schedule**
   - Meet every Wednesday after class or whenever additional discussion is needed.

2. **Communication Platform**
   - Discord Group, Messenger Group, Telegram group for daily communication.
   - GitHub Issues for tracking project tasks.

3. **Task Assignment**
   - Responsibilities are divided fairly among all team members based on their strengths and current project needs.
   - All members are expected to contribute through commits, documentation updates, diagrams, or issue management.

4. **File Naming Convention**
   - Use lowercase letters with hyphens.
   - Example: `user-persona.md`

5. **GitHub Commit Rule**
   - Make meaningful commits after completing each task.
   - Example:
     - `Add user persona document`
     - `Create user flow diagram`
     - `Update MVP feature list`

6. **Issue Management**
   - Create a GitHub Issue before starting major tasks.
   - Close the issue only after the task has been completed and committed.

7. **Conflict Resolution**
   - Discuss disagreements as a team first.
   - If no agreement is reached, decide by majority vote.
   - If necessary, seek guidance from the instructor.

8. **Quality Assurance**
   - Review all documents before committing.
   - Ensure every file follows the course template and is consistent with previous lab work.
