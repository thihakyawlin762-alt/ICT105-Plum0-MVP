# Lab 04 - User Persona

## 1. Persona Name

Aung Min, Second-Year University Student

---

## 2. User Type

University student who manages multiple courses and frequently struggles to keep track of assignment deadlines from different platforms.

---

## 3. Background and Context

### Where does this user experience the problem?

- During the academic semester while attending university classes.
- When lecturers post assignments on different platforms such as LMS, Microsoft Teams, email, and class chat groups.
- While managing several subjects with different submission dates.

### When does the problem usually happen?

- During busy weeks with multiple assignments.
- Before assignment submission deadlines.
- During midterm and final examination periods.
- Whenever lecturers announce deadlines through different communication channels.

### What digital tools does this user already use?

- University LMS
- Microsoft Teams
- Email
- LINE / WhatsApp / Messenger class groups
- Google Calendar (occasionally)
- Phone Reminder
- Personal notebook or sticky notes

---

## 4. Goals

- Never miss assignment deadlines.
- View all assignments in one place.
- Receive reminders before deadlines.
- Organize coursework more efficiently.
- Reduce stress caused by forgotten assignments.
- Improve time management and academic performance.

---

## 5. Pain Points from Lab 03 Evidence

| Pain Point | Evidence ID / Respondent | Explanation |
|---|---|---|
| Assignment information is scattered across multiple platforms. | R1, R2, R4, R5 | Students need to check several apps every day to avoid missing announcements. |
| Students forget assignment deadlines. | R1, R3, R5 | Important deadlines are forgotten during busy academic weeks. |
| Reminder methods are inconsistent. | R2, R4 | Some lecturers announce through LMS while others use chat groups or email. |
| Students spend too much time searching for assignment information. | R2, R3, R5 | They repeatedly switch between applications to verify deadlines. |
| Managing multiple courses becomes confusing. | R1, R4 | Students struggle to organize deadlines from several subjects simultaneously. |

---

## 6. Current Alternatives / Workarounds

| Current Alternative | Weakness / Limitation | Evidence |
|---|---|---|
| Personal notebook | Easy to forget updating manually. | R1, R3 |
| Phone reminders | Must be entered manually for every assignment. | R2, R5 |
| Google Calendar | Requires manual setup and regular maintenance. | R2 |
| Checking LMS frequently | Time-consuming and inefficient. | R1, R4 |
| Reading class chat groups | Important messages are easily buried by new conversations. | R3, R5 |

---

## 7. Design Implications

### What should the system support?

- Display all assignments in one dashboard.
- Allow students to add and edit assignment information.
- Organize assignments by subject.
- Show upcoming deadlines clearly.
- Send reminder notifications before due dates.
- Allow filtering by completed and pending assignments.
- Provide a simple and mobile-friendly interface.

### What should the team avoid?

- Complicated navigation.
- Too many unnecessary features.
- Difficult setup process.
- Excessive manual data entry.
- Confusing interface design.

### Which user need is most important for the MVP?

The MVP should primarily help students organize all assignment deadlines in one place and remind them before submission dates so they can avoid missing deadlines.
