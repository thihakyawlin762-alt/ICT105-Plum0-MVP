# Experiment Script

## 1. Opening Script

Thank you for helping us test our Smart Assignment Tracker prototype.

This is not a test of your ability. We are testing whether our product idea, interface, and workflow are clear and easy to use.

While using the prototype, please think aloud by telling us what you expect to happen, what is confusing, and what you find easy or difficult.

---

## 2. Test Scenario

You are a university student who wants to organize your coursework using the Smart Assignment Tracker.

You have several assignments from different courses and want to manage deadlines, monitor your progress, and keep your assignments organized in one place.

---

## 3. Tasks for Tester

| Task No. | Task Instruction                                                    | Related Requirement | Expected Result                                                 |
| -------- | ------------------------------------------------------------------- | ------------------- | --------------------------------------------------------------- |
| T1       | Open the homepage and explain what this system is designed to do.   | FR-01               | User understands the purpose of the system.                     |
| T2       | Log into the Smart Assignment Tracker.                              | FR-02               | User successfully accesses the dashboard.                       |
| T3       | Add a new assignment using the assignment form.                     | FR-03, FR-10, FR-11 | User completes the form and receives a confirmation message.    |
| T4       | View the assignment list and search for the assignment you created. | FR-05, FR-06        | User successfully locates the assignment.                       |
| T5       | Open the assignment details page.                                   | FR-07               | User understands the assignment information and current status. |
| T6       | Edit the assignment information and save the changes.               | FR-04               | User successfully updates the assignment.                       |
| T7       | View the dashboard summary.                                         | FR-12               | User understands the dashboard summary and assignment overview. |

---

## 4. Observation Notes

During the experiment, observe and record:

* Where the tester hesitates before taking an action.
* Any incorrect clicks or navigation mistakes.
* Questions asked by the tester.
* Features that appear confusing.
* Features that users find helpful.
* Suggestions for improving the interface or workflow.

---

## 5. Closing Questions

1. What part of the Smart Assignment Tracker was the easiest to use?
2. Which task was the most confusing or difficult?
3. Did the dashboard help you understand your assignments?
4. Would you use this system to manage your real coursework? Why or why not?
5. Which feature did you find the most useful?
6. What improvements would you suggest before the final prototype is completed?
