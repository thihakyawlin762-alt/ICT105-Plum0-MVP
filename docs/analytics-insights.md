# Lab 08 - Analytics Insights

## 1. Analytics Question

What did users do, say, and prove when testing the MVP direction?

The validation aimed to determine whether students and lecturers could successfully use the Smart Assignment Tracker to manage assignments efficiently. The experiment focused on task completion, ease of use, feature usefulness, user interest, and identifying usability issues before continuing implementation.

---

## 2. Metrics Calculated

| Metric | Formula / Method | Result |
|---|---|---|
| Total test users | Count testers | 20 |
| Completed tasks | Count Task_Completed = Yes | 18 |
| Task success rate | Completed tasks / total testers | 90% |
| Average feedback score | Average EaseOfUseScore | 4.3 / 5 |
| Average interest level | Average InterestLevel | 4.5 / 5 |
| Confusion points | Count repeated confusion categories | Search filters, Save button, Validation message, Navigation menu |

---

## 3. Findings

1. **90% of participants successfully completed their assigned tasks**, showing that the prototype supports the main assignment management workflow.

2. **Users rated the Assignment Form, Dashboard, Search Function, and Assignment Details as the most useful features**, indicating that these core functions provide strong user value.

3. **Most usability issues were related to interface clarity rather than missing functionality**, such as search filter labels, button placement, and validation messages.

---

## 4. Interpretation

The experiment shows that the Smart Assignment Tracker MVP is able to support the primary needs of students and lecturers. Most users completed their tasks without assistance and provided positive feedback. The remaining issues are mainly usability improvements, meaning the project is suitable to continue into further implementation with minor interface refinements.
