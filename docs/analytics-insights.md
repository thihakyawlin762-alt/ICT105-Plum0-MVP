# Lab 08 - Analytics Insights

## 1. Analytics Question

What did users do, say, and prove when testing the MVP direction?

The purpose of the customer validation was to evaluate whether students and lecturers could successfully use the Smart Assignment Tracker to manage assignments efficiently. The validation focused on task completion, ease of use, user interest, feature usefulness, and identifying usability issues that should be improved before final implementation.

---

## 2. KPI Results

| Metric | Formula / Method | Result |
|---|---|---|
| Total Tests | Count of validation sessions | **20** |
| Task Success Rate | Completed Tasks / Total Tests | **95%** |
| Average Ease of Use | Average EaseOfUseScore | **4.35 / 5** |
| Interest Rate | Users who would use the system again / Total Tests | **95%** |
| Main Confusion Points | Most frequently reported issues | Search filters, Save button, Navigation menu, Date picker |

---

## 3. Key Findings

1. **95% of users successfully completed their assigned tasks**, indicating that the Smart Assignment Tracker supports the primary assignment management workflow effectively.

2. **Users rated the system highly for ease of use, with an average score of 4.35 out of 5**, suggesting that the interface is generally intuitive and easy to navigate.

3. **The Interest Rate of 95% shows strong user acceptance**, meaning that most participants indicated they would continue using the system in the future.

4. **Most users experienced no major usability problems.** The few issues reported were mainly related to interface elements such as search filter labels, button placement, navigation, and the date picker rather than missing system functionality.

---

## 4. Interpretation

The customer validation results indicate that the Smart Assignment Tracker MVP achieved its primary objectives. Most participants successfully completed their assigned tasks, rated the system highly for ease of use, and expressed strong interest in using it again. The feedback suggests that the MVP is functionally effective, while the remaining issues are minor usability improvements rather than major design or functionality problems. Overall, the validation provides strong evidence that the project is ready to continue toward the final prototype with targeted interface refinements.

---

## 5. Requirements Affected

| Requirement ID | Validation Evidence | Recommended Action |
|---|---|---|
| FR-05 | Some users found search filter labels unclear. | Improve search filter labels and filtering options. |
| FR-07 | A few users required additional time to locate the Save button. | Increase button visibility and improve layout consistency. |
| FR-04 | Validation messages were not immediately noticeable. | Make validation messages more visible and easier to understand. |
| FR-11 | Some users experienced minor navigation difficulties. | Simplify dashboard navigation and improve menu organization. |

---

## 6. Prototype Improvement Actions

Based on the customer validation results, the following improvements are recommended before the final implementation:

- Improve search filter labels and keyword search functionality.
- Increase the visibility of important action buttons such as **Save** and **Edit**.
- Enhance validation messages for required fields and user feedback.
- Simplify dashboard navigation to improve usability.
- Improve interface consistency and mobile responsiveness.
- Continue implementing the remaining lecturer-side features before the final release.

---

## 7. Overall Conclusion

The customer validation demonstrates that the Smart Assignment Tracker MVP is **largely validated**. With a **95% Task Success Rate**, **4.35/5 Average Ease of Use**, and **95% Interest Rate**, users showed strong acceptance of the system. The remaining issues are minor usability improvements that can be addressed before the final prototype is completed.
