# Lab 09 Data Handling Policy

## Data Collection

The Smart Assignment Tracker prototype collects only the information necessary to manage assignments and evaluate the system during user testing.

Data collected includes:

- Student ID
- Student Name
- User Role (Student or Lecturer)
- Assignment Title
- Module / Course
- Assignment Description
- Due Date
- Assignment Status
- Lecturer Feedback
- Validation test results (feedback score, interest level, task completion, completion time)

## Data Storage

During prototype development, data will be stored in:

- CSV files for validation and analytics
- Microsoft Excel spreadsheets for validation analysis
- GitHub repository for project documentation
- Local project files during development

No real production database is used in this prototype.

## Data Access

Access to project data is limited to:

- Project team members
- Course instructor
- Teaching assistants (if required)

Lecturer features are only accessible to lecturer users, while student features are limited to students.

## Data Minimization

The following information is intentionally NOT collected because it is unnecessary:

- Home address
- Phone number
- National ID / Passport number
- Bank account information
- Payment information
- Personal photographs unrelated to assignments
- Other sensitive personal information

Only the minimum academic information required for assignment management is collected.

## Responsible Data Rule

The Smart Assignment Tracker follows responsible data practices by collecting only necessary academic information. Sensitive personal data is avoided whenever possible. Sample or masked data is used during testing and validation activities. User information is handled only for educational purposes within the ICT105 project and is not shared outside the project team.
