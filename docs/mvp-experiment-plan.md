## 2. Experiment Objective

We want to evaluate whether university students can successfully complete the core assignment management workflow using the Smart Assignment Tracker prototype without assistance and determine whether the interface is intuitive and useful for everyday academic task management.

## 4. MVP Experiment Type

### Selected Experiment Type

* Simple Web Prototype
* Clickable Prototype

### Reason for Selection

The Smart Assignment Tracker already has an interactive HTML/CSS/JavaScript prototype that allows users to navigate between pages, add assignments, edit assignments, search records, and view dashboards. Using the existing prototype enables realistic usability testing without requiring a complete backend implementation.

## 6. Experiment Procedure Summary

Each participant will use the Smart Assignment Tracker prototype individually.

Participants will be asked to:

1. Open the homepage.
2. Log into the system.
3. Add a new assignment.
4. View the assignment list.
5. Search for an assignment.
6. Open the assignment details.
7. Edit the assignment.
8. View the dashboard summary.
9. Complete the feedback form after finishing the tasks.

## 7. Expected Learning

After completing the experiment, the team expects to determine:

* Whether students can complete the core workflow without assistance.
* Which interface elements are confusing or difficult to use.
* Which features provide the greatest value to users.
* Which improvements should be prioritized before the next implementation stage.
* Whether the current MVP is suitable for continued development and user validation.

During the experiment, the team will observe task completion, navigation behavior, errors, confusion points, and overall user experience.
