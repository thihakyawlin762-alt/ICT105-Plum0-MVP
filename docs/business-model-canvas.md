# Business Model Canvas

## Project Title

Smart Assignment Tracker

## 1. Customer Segments

Primary Users

* University students who need to manage assignments, deadlines, and academic tasks.

Secondary Users

* Lecturers/Administrators who monitor assignment records, update assignment status, and manage assignment information.

---

## 2. Value Proposition

The Smart Assignment Tracker helps university students organize assignments in one place, monitor deadlines, receive reminders, and track completion status. It reduces missed deadlines, improves academic organization, and provides lecturers with a simple way to oversee assignment records.

---

## 3. Channels

Users can access the Smart Assignment Tracker through:

* University web browser
* Desktop or laptop computers
* Mobile web browsers
* University Learning Management System (future integration)
* GitHub Pages (prototype demonstration)

---

## 4. Customer Relationships

The system maintains interaction with users by providing:

* Secure student and lecturer login
* Personalized dashboard
* Assignment reminders
* Assignment status tracking
* Easy navigation and self-service management
* Lecturer monitoring and assignment management

---

## 5. Revenue Streams / Sustainability Logic

The Smart Assignment Tracker is not intended to generate direct revenue.

Instead, it provides value by:

* Improving students' time management
* Reducing missed assignment deadlines
* Increasing academic productivity
* Helping lecturers monitor assignment progress more efficiently
* Supporting universities with a centralized assignment management system

---

## 6. Key Resources

The project requires:

* HTML, CSS, and JavaScript
* LocalStorage and sample assignment data
* Web browser
* GitHub Repository
* GitHub Pages (prototype hosting)
* Wireframes and UI design
* Project documentation
* Development team members

---

## 7. Key Activities

The system performs the following activities:

* User login
* Assignment creation
* Assignment editing
* Assignment deletion
* Assignment search and filtering
* Assignment detail viewing
* Assignment status updates
* Reminder display
* Dashboard summary generation
* Lecturer assignment management

---

## 8. Key Partners

Potential partners include:

* University
* Lecturers
* Students
* ICT105 Course Instructor
* GitHub
* Web browser providers

---

## 9. Cost Structure

Possible project costs include:

* Development time
* Testing and debugging
* Internet connection
* Computer equipment
* GitHub hosting (Free)
* Future web hosting or domain (if deployed)
* System maintenance and future improvements

---

## Final Note

This Business Model Canvas directly supports the Smart Assignment Tracker MVP. Every business component aligns with the Functional Requirements (FR), User Stories (US), Feature Requirement Mapping, Wireframes, User Flows, and the HTML/CSS/JavaScript prototype. The Business Model Canvas ensures that the final prototype delivers value to students and lecturers while remaining practical within the scope of this semester's project.
