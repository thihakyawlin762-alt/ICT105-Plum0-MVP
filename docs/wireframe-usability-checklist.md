# Wireframe Usability Review Checklist

Use this checklist before submitting Lab 05 to ensure that the Smart Assignment Tracker wireframes are complete, consistent, and aligned with the system requirements.

| Check Item | Yes/No | Evidence or Screenshot |
|---|---|---|
| The homepage clearly introduces the Smart Assignment Tracker, target users, and provides a clear Login button. | Yes | Homepage wireframe |
| The main user pathway can be followed from Login → Dashboard → Add Assignment → View/Edit Assignment → Logout. | Yes | User Flow Diagram |
| The Add Assignment form includes all required fields, validation messages, and confirmation feedback. | Yes | Input Form wireframe |
| The assignment list includes search, filter, sorting options, assignment status, and priority indicators. | Yes | Record List wireframe |
| The assignment detail page displays complete assignment information and current status. | Yes | Detail View wireframe |
| The Lecturer/Administrator page provides assignment management functions such as view, edit, delete, and update status. | Yes | Admin View wireframe |
| The dashboard displays meaningful summary information such as total assignments, completed assignments, pending assignments, and upcoming deadlines. | Yes | Dashboard wireframe |
| Navigation is consistent across all wireframes (navigation bar, buttons, labels, and layout). | Yes | All wireframes |
| All wireframes use realistic sample assignment data instead of empty placeholder boxes. | Yes | All wireframes |
| Every wireframe can be traced back to the corresponding Functional Requirements (FR) and User Stories (US) from Lab 04. | Yes | Feature-Requirement Mapping |
| The overall interface is simple, easy to understand, and suitable for university students. | Yes | Complete wireframe set |
| The wireframes closely match the planned HTML/CSS prototype for the final project. | Yes | Prototype comparison |
